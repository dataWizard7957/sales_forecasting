# SuperKart Sales Forecasting

Machine learning system for forecasting product sales across SuperKart retail outlets.  
The project predicts product-level revenue using historical product and store attributes to support inventory planning and regional sales strategy.

---

# Business Context

Sales forecasting is the process of predicting future sales revenue based on historical data, market trends, and operational insights.

Accurate forecasting helps businesses:

- Plan inventory procurement
- Optimize supply chain operations
- Improve regional sales planning
- Reduce forecasting risks
- Support strategic business decisions

SuperKart is a retail chain operating supermarkets and food marts across various tier cities.  
To optimize inventory management and improve sales planning, SuperKart wants to forecast **product-level sales revenue for each store**.

This project develops a **machine learning forecasting solution** that can be integrated into SuperKart's decision-making systems.

---

# Objective

Build a machine learning system that predicts:

```
Product_Store_Sales_Total
```

This represents the **total sales revenue generated by a product in a specific store**.

The solution includes:

- Data preprocessing pipeline
- Machine learning regression models
- Model evaluation and comparison
- Model serialization
- API deployment
- Containerized deployment using Docker

---

# Dataset Description

The dataset contains attributes related to **products and stores**.

Target Variable:

```
Product_Store_Sales_Total
```

This variable represents the **total revenue generated by the sale of a product in a particular store**.

---

# Data Dictionary

| Feature | Description |
|------|------|
| Product_Id | Unique identifier for each product |
| Product_Weight | Weight of each product |
| Product_Sugar_Content | Sugar level (Low Sugar, Regular, No Sugar) |
| Product_Allocated_Area | Ratio of display area allocated to the product |
| Product_Type | Product category (meat, snacks, dairy, etc.) |
| Product_MRP | Maximum retail price |
| Store_Id | Unique identifier of store |
| Store_Establishment_Year | Year store was established |
| Store_Size | Store size (High, Medium, Low) |
| Store_Location_City_Type | Tier classification of city (Tier 1, 2, 3) |
| Store_Type | Store format (Departmental Store, Supermarket, Food Mart) |
| Product_Store_Sales_Total | Total revenue generated by the product |

# Setup

Clone the repository:

```bash
git clone https://github.com/<your-username>/sales-forecasting.git
cd sales_prediction2
```

Create virtual environment:

```bash
python -m venv venv
```

Activate environment

Linux / Mac

```bash
source venv/bin/activate
```

Windows

```bash
venv\Scripts\activate
```

Install dependencies

```bash
pip install -r requirements.txt
```

---

# Training the Model

Run the training script:

```bash
python model_building/train.py
```

This script will:

1. Load dataset
2. Perform preprocessing
3. Train regression models
4. Evaluate performance
5. Save the trained model

---

# Running the API

Start the backend API:

```bash
python deployment/app.py
```

The API will expose endpoints for **sales prediction requests**.

---

# Deployment

The project supports **containerized deployment using Docker**.

Build Docker image:

```bash
docker build -t superkart-forecast deployment/
```

Run container:

```bash
docker run -p 5000:5000 superkart-forecast
```
